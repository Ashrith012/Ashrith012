<br/>

A multi-agent LLM orchestration platform coordinating specialized agents for SQL querying, RAG retrieval,
knowledge-base lookups, chart generation, and report writing — unified through a Supervisor router and a
Planner-Executor-Combiner pipeline for cross-database intelligence.

| Aspect | Details |
|---|---|
| **Stack** | Python · LangGraph · FastAPI · React · Google Gemini · ChromaDB |
| **Scale** | 7 specialized agents across MySQL & MSSQL data sources |
| **Performance** | Real-time AG-UI SSE streaming with live agent-step visualization |
| **Security** | Isolated agent scopes, controlled cross-database query routing |
| **Impact** | Automated cross-source reporting via auto-generated charts & PDF/DOCX docs |
| **Repository** | `🔗 link-to-repo` |

**Engineering highlights:**
- Supervisor-router architecture directing 7 specialized agents through a coordinated pipeline
- FastAPI backend with AG-UI SSE streaming, file/Google Drive ingestion, and PDF/DOCX generation
- React frontend rendering real-time agent execution steps and auto-generated visual analytics
- Langfuse-based LLM observability integrated across the full agent pipeline
- ChromaDB vector store auto-synced with a WordPress CMS for a continuously updated knowledge base
- PandasAI layered on top of live query results for enhanced analytical depth

</details>

<details>
<summary><b>🛣️ Infra DMT POC — Highway Earthwork Volume Calculation Engine</b></summary>
<br/>

A Streamlit-based engineering analysis system (XDOT Road Quantity Analyzer) that extracts vector geometry
from CAD-generated highway engineering PDFs to automatically compute cut/fill earthwork volumes.

| Aspect | Details |
|---|---|
| **Stack** | Python · PyMuPDF · Streamlit · NumPy |
| **Scale** | Six-stage pipeline across classification, splitting, and volume calculation |
| **Performance** | Vector-based (non-pixel) geometry extraction for high-fidelity parsing |
| **Security** | Deterministic, auditable geometry-to-volume computation trail |
| **Impact** | Automates manual highway earthwork quantity takeoffs from engineering drawings |
| **Repository** | `🔗 link-to-repo` |

**Engineering highlights:**
- Six-stage pipeline: PDF classification → cross-section splitting → geometry extraction → profile identification → coordinate transformation → volume calculation
- Custom `cross_section_detector.py` filtering cross-sections by station-series prefix (19-series / 23-series)
- Triple-guard grid filter system in `profile_identifier.py` to reduce Proposed Grade misidentification
- Volume computation via the Average End Area method
- Full orchestrator audit ensuring correct region-bound passing and coordinate-transform validation

</details>
